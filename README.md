# ಶಾಲೆ ಮೌಲ್ಯಮಾಪನ ವ್ಯವಸ್ಥೆ (Shale Moulyamapana Vyavasthe)

ವಿದ್ಯಾರ್ಥಿಗಳ ಅಂಕಗಳು, ಹಾಜರಾತಿ ಮತ್ತು ವರದಿಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಇರುವ ಒಂದೇ-ಫೈಲ್ (single-file) ವೆಬ್ ಅಪ್ಲಿಕೇಶನ್. ಇದನ್ನು ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನೇರವಾಗಿ ತೆರೆಯಬಹುದು, ಅಥವಾ GitHub Pages ಮೂಲಕ ಆನ್‌ಲೈನ್‌ನಲ್ಲಿ ಹೋಸ್ಟ್ ಮಾಡಬಹುದು.

## ವೈಶಿಷ್ಟ್ಯಗಳು
- ವಿದ್ಯಾರ್ಥಿಗಳ ಪಟ್ಟಿ, ಅಂಕಗಳು (FA/SA), ಹಾಜರಾತಿ ನಿರ್ವಹಣೆ
- ಶಿಕ್ಷಕರ ಖಾತೆಗಳು ಮತ್ತು ವಿಷಯ ಹಂಚಿಕೆ
- ವರದಿ ಮತ್ತು ಅಂಕಪಟ್ಟಿ (marksheet) ಮುದ್ರಣ/ಎಕ್ಸ್‌ಪೋರ್ಟ್
- Firebase Realtime Database ಬಳಸಿ ಐಚ್ಛಿಕ ಆನ್‌ಲೈನ್ ಸಿಂಕ್ (ಇಲ್ಲದಿದ್ದರೆ ಆಫ್‌ಲೈನ್ ಮೋಡ್‌ನಲ್ಲಿ ಕೆಲಸ ಮಾಡುತ್ತದೆ)

## ಬಳಕೆ
`index.html` ಫೈಲ್ ಅನ್ನು ಯಾವುದೇ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ತೆರೆದರೆ ಸಾಕು — ಸರ್ವರ್ ಅಗತ್ಯವಿಲ್ಲ.

## GitHub Pages ಮೂಲಕ ಹೋಸ್ಟ್ ಮಾಡುವುದು
1. ಈ ಫೈಲ್‌ಗಳನ್ನು ಒಂದು GitHub repository ಗೆ ಅಪ್‌ಲೋಡ್ ಮಾಡಿ
2. Repository → **Settings → Pages** ಗೆ ಹೋಗಿ
3. **Source**: `main` branch, `/ (root)` ಆಯ್ಕೆ ಮಾಡಿ → **Save**
4. ಕೆಲವು ನಿಮಿಷಗಳಲ್ಲಿ `https://<your-username>.github.io/<repo-name>/` ಎಂಬ ಲಿಂಕ್‌ನಲ್ಲಿ ಸೈಟ್ ಲೈವ್ ಆಗುತ್ತದೆ

## Firebase ಸಿಂಕ್ (ಐಚ್ಛಿಕ)
ಅನೇಕ ಶಿಕ್ಷಕರು ಒಂದೇ ಡೇಟಾವನ್ನು ಹಂಚಿಕೊಳ್ಳಬೇಕಿದ್ದರೆ, ಅಪ್ಲಿಕೇಶನ್‌ನ "☁️ Firebase Sync" ಸೆಟ್ಟಿಂಗ್‌ನಲ್ಲಿ ನಿಮ್ಮ ಸ್ವಂತ Firebase Realtime Database ಕಾನ್ಫಿಗ್ ಹಾಕಬಹುದು. ಒಮ್ಮೆ ಹಾಕಿ ಉಳಿಸಿದರೆ ಸಾಕು — ಮುಂದಿನ ಬಾರಿ ಆಪ್ ತೆರೆದಾಗ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಮತ್ತೆ ಸಂಪರ್ಕವಾಗುತ್ತದೆ, ಮತ್ತೆ ಕೀ ಕೇಳುವುದಿಲ್ಲ.

## 📱 PWA ಆಗಿ ಫೋನ್/ಕಂಪ್ಯೂಟರ್‌ಗೆ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುವುದು
ಈ ಆಪ್ ಒಂದು Progressive Web App (PWA) — ಇದನ್ನು ಪ್ಲೇ ಸ್ಟೋರ್ ಇಲ್ಲದೆಯೇ ಫೋನ್/ಕಂಪ್ಯೂಟರ್‌ನಲ್ಲಿ ಸಾಮಾನ್ಯ ಆಪ್‌ನಂತೆ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬಹುದು (ಆಫ್‌ಲೈನ್‌ನಲ್ಲೂ ತೆರೆಯುತ್ತದೆ, ಪ್ರತ್ಯೇಕ ಐಕಾನ್ ಸಿಗುತ್ತದೆ).

**ಷರತ್ತು:** GitHub Pages ಲಿಂಕ್ (https://) ಮೂಲಕವೇ ತೆರೆಯಬೇಕು — PWA ಇನ್‌ಸ್ಟಾಲ್‌ಗೆ HTTPS ಅಗತ್ಯ.

- **Android (Chrome):** ಸೈಟ್ ತೆರೆದು ⋮ ಮೆನುನಲ್ಲಿ **"Install app"** ಅಥವಾ **"Add to Home screen"** ಒತ್ತಿ
- **iPhone (Safari):** ಸೈಟ್ ತೆರೆದು Share ಬಟನ್ → **"Add to Home Screen"** ಒತ್ತಿ
- **ಕಂಪ್ಯೂಟರ್ (Chrome/Edge):** ಅಡ್ರೆಸ್ ಬಾರ್‌ನ ಬಲಭಾಗದಲ್ಲಿ ಇನ್‌ಸ್ಟಾಲ್ ಐಕಾನ್ (⊕/💻) ಒತ್ತಿ

ಇನ್‌ಸ್ಟಾಲ್ ಆದ ನಂತರ ಆಪ್ ಪ್ರತ್ಯೇಕವಾಗಿ (ಬ್ರೌಸರ್ ಬಾರ್ ಇಲ್ಲದೆ) ತೆರೆಯುತ್ತದೆ, ಮತ್ತು ಈ ಡಿವೈಸ್‌ನಲ್ಲಿ ಒಮ್ಮೆ Firebase ಕೀ ಹಾಕಿದ ನಂತರ ಮತ್ತೆ ಕೇಳುವುದಿಲ್ಲ — ಡೇಟಾ ತಾನಾಗಿಯೇ ಸಿಂಕ್ ಆಗುತ್ತಲೇ ಇರುತ್ತದೆ.

**ಗಮನಿಸಿ:** `sw.js` ಫೈಲ್‌ನಲ್ಲಿ `CACHE_NAME` ಅನ್ನು (ಉದಾ: `v1` → `v2`) ಬದಲಿಸದೆ ಹೊಸ `index.html` ಅಪ್‌ಲೋಡ್ ಮಾಡಿದರೆ, ಈಗಾಗಲೇ ಇನ್‌ಸ್ಟಾಲ್ ಆದ ಬಳಕೆದಾರರಿಗೆ ಹಳೆಯ ಕ್ಯಾಶ್ ಆವೃತ್ತಿ ಸ್ವಲ್ಪ ಸಮಯ ಕಾಣಿಸಬಹುದು. ಆದ್ದರಿಂದ ಆಪ್ ಅಪ್‌ಡೇಟ್ ಮಾಡಿದಾಗಲೆಲ್ಲ `sw.js` ನಲ್ಲಿ `CACHE_NAME` ಸಂಖ್ಯೆ ಹೆಚ್ಚಿಸಿ.
